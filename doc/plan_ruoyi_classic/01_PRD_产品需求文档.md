# 文瀛餐厅管理系统 — 产品需求文档

## 1. 项目背景

高校食堂日常运营存在菜品维护分散、库存盘点依赖人工、订单数据统计不统一、经营分析不足的问题。本项目交付一个可运行的后台管理系统，用于支撑答辩演示和基础业务闭环。

系统聚焦后台管理端，不开发真实点餐小程序。前端点餐场景使用后台“模拟下单”完成演示闭环。

## 2. 项目目标

- 完成菜品、食材、配方、库存、订单、看板六类核心业务闭环
- 支持三角色登录和菜单权限隔离
- 支持“完成订单后按配方扣减库存”的核心业务演示
- 保证 10 天内可实现、可联调、可答辩

## 3. 用户角色

| 角色 | 代码 | 核心职责 |
|------|------|----------|
| 系统管理员 | ADMIN | 用户、角色、菜单、基础配置 |
| 食堂经理 | MANAGER | 菜品、配方、订单、看板 |
| 库管员 | STOCK_KEEPER | 食材、入库、库存盘点 |

## 4. 功能范围

### 4.1 菜品管理

- 菜品分类增删改查
- 菜品增删改查
- 菜品上下架
- 按分类、名称、状态筛选

### 4.2 食材与库存

- 食材档案增删改查
- 新增食材时自动初始化库存记录
- 入库登记并自动累加库存
- 库存列表与低库存预警

### 4.3 菜品配方

- 为菜品配置食材与单份用量
- 按菜品查看当前配方
- 覆盖保存配方

### 4.4 订单管理

- 后台模拟下单
- 订单列表和订单详情
- 完成订单
- 退单

### 4.5 数据看板

- 今日营收
- 今日订单数
- 近 7 天营收趋势
- 菜品销量 Top10
- 库存预警列表

## 5. 明确不做

- 真实点餐小程序
- 在线支付
- Docker 部署
- Excel 导出
- 图片上传
- 供应商完整模块
- 同环比分析
- 理论消耗对比报表
- 多仓库、多档口复杂场景

## 6. 非功能要求

- PC 浏览器可正常使用，推荐 Chrome/Edge
- 列表查询响应时间目标小于 1 秒
- 看板接口响应目标小于 2 秒
- 采用若依内置 JWT、角色权限、操作日志能力
- 安装说明应能支撑空环境快速跑通

## 7. 验收标准

1. 三角色均可登录，菜单权限隔离生效
2. 可完成“配方维护 -> 食材入库 -> 模拟下单 -> 完成订单 -> 库存自动扣减”演示闭环
3. 数据看板图表正常显示，统计结果与订单数据一致
4. 低库存食材有明显预警展示
5. 项目具备可执行的安装使用说明
