# 文瀛餐厅管理系统 — 产品需求文档 (PRD)
## 方案 A：若依 RuoYi-Vue + Java

**版本**：v1.0  **日期**：2026-06-14  **状态**：待评审

---

## 1. 项目背景

高校食堂"文瀛餐厅"日常运营存在四个痛点：

| 痛点 | 现状 | 目标 |
|------|------|------|
| 菜品管理粗放 | 纸质记录，易出错 | 系统化管理上下架、改价 |
| 库存靠人工盘点 | 损耗无法量化 | 按配方自动扣减，实时掌握库存 |
| 订单数据割裂 | 前端点餐与后台脱节 | 统一录入、统计、分析 |
| 缺乏数据决策 | 备餐凭经验 | 可视化看板辅助决策 |

本系统为**后台管理端**，以若依（RuoYi-Vue）为脚手架底座，10 天内交付。

---

## 2. 用户角色

| 角色 | 代码 | 核心职责 |
|------|------|---------|
| 系统管理员 | ADMIN | 用户管理、权限配置、字典维护 |
| 食堂经理 | MANAGER | 菜品管理、订单查看、数据看板 |
| 库管员 | STOCK_KEEPER | 食材档案、入库、库存盘点 |

> 三个角色通过若依内置 RBAC 控制菜单可见性，无需额外开发。

---

## 3. 功能需求清单

### 3.1 菜品管理（MANAGER）

| # | 功能 | 描述 | 优先级 |
|---|------|------|--------|
| F01 | 菜品分类 | 增删改查分类（主食/热菜/凉菜/汤品/饮品） | P0 |
| F02 | 菜品列表 | 分页查询，支持按分类、名称筛选 | P0 |
| F03 | 新增/编辑菜品 | 名称、分类、价格、描述、状态 | P0 |
| F04 | 上架/下架 | 一键切换售卖状态 | P0 |
| F05 | 删除菜品 | 逻辑删除（del_flag） | P0 |

### 3.2 食材管理（STOCK_KEEPER）

| # | 功能 | 描述 | 优先级 |
|---|------|------|--------|
| F06 | 食材档案 | 增删改查（名称、单位、安全库存阈值） | P0 |
| F07 | 入库登记 | 记录采购数量、单价，自动累加库存 | P0 |
| F08 | 库存列表 | 展示当前库存量，低于安全值标红预警 | P0 |

### 3.3 菜品配方（MANAGER）

| # | 功能 | 描述 | 优先级 |
|---|------|------|--------|
| F09 | 配方维护 | 为每道菜配置消耗食材及单份用量 | P0 |
| F10 | 配方查看 | 列表展示菜品对应的食材配方 | P0 |

### 3.4 订单管理（MANAGER）

| # | 功能 | 描述 | 优先级 |
|---|------|------|--------|
| F11 | 模拟下单 | 选择菜品+数量，生成订单 | P0 |
| F12 | 订单列表 | 分页查询，支持按状态/时间筛选 | P0 |
| F13 | 订单详情 | 展示订单明细 | P0 |
| F14 | 完成订单 | 状态变更为"已完成"，**触发库存自动扣减** | P0 |
| F15 | 退单 | 状态变更为"已退单"，不扣库存 | P1 |

### 3.5 数据看板（MANAGER）

| # | 功能 | 描述 | 优先级 |
|---|------|------|--------|
| F16 | 今日营收 | 数字卡片 | P0 |
| F17 | 今日订单数 | 数字卡片 | P0 |
| F18 | 近 7 天营收趋势 | 折线图 | P0 |
| F19 | 菜品销量 Top10 | 横向柱状图 | P0 |
| F20 | 库存预警提示 | 低库存食材列表（嵌入看板） | P1 |

---

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 列表查询 < 1s；看板统计 < 2s |
| 安全 | JWT 鉴权；密码 BCrypt 加密；MyBatis 预编译防 SQL 注入 |
| 兼容 | Chrome/Edge 最新版；PC 1280px 以上分辨率 |
| 自主可控 | 全栈开源；MySQL 可迁移 openGauss/达梦 |
| 演示 | 内置 30 菜品 + 200 条近 7 天订单演示数据 |

---

## 5. 不在范围内（Out of Scope）

- 前端点餐小程序（预留接口，不开发）
- 在线支付
- 图片上传（菜品图片用 URL 字符串）
- Excel 导出
- 供应商档案
- 同环比分析、食材理论消耗 vs 实际对比分析

---

## 6. 验收标准

1. 三个角色均可正常登录，菜单权限互相隔离
2. 完整演示"配方录入 → 食材入库 → 模拟下单 → 完成订单 → 库存自动扣减"全闭环
3. 数据看板 4 张图正常渲染，数字与订单数据一致
4. 库存低于安全阈值时红色预警显示
