> 策略：**站在若依脚手架肩膀上**，RBAC/JWT/代码生成免费得，自己只写业务逻辑。

## 核心变化（对比初版 MVP）

| 方面 | 初版 MVP（过度简化） | 本版修订 |
|------|---------------------|---------|
| 框架 | 从零搭 | **若依 RuoYi-Vue 底座**，省 40% 工作量 |
| 角色 | 单管理员 | **3 角色**（管理员/经理/库管员），若依 RBAC 配置即可 |
| 库存 | 菜品自带 stock 字段 | **食材表+配方表+库存表**，完整联动 |
| 表数量 | 4 张 | **8 张业务表**（+若依自带 sys_* 表） |
| 看板 | 4 图 | **4 图**（不变，够用） |

## 三句话总结

1. **用若依**：RBAC/JWT/代码生成器都有了，不用自己造轮子
2. **保留配方联动**：菜品→配方→食材→库存，订单完成自动扣减，这是核心价值
3. **演示路线**：登录三角色 → 配方维护 → 入库 → 模拟下单→完成→看库存变化 → 看板
