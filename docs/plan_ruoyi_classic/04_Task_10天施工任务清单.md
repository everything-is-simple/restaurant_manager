# 文瀛餐厅管理系统 — 10天施工任务清单

## 1. 总原则

- 唯一底座：`RuoYi-Vue 3.9.2` 经典版
- 先跑通环境，再做代码生成，再补业务逻辑
- 优先主闭环，不为边角功能分散时间

## 2. 10 天排期

### D1 环境准备

- 安装并验证 JDK 17、Maven、MySQL、Redis、Node 16
- 导入若依初始化 SQL
- 导入业务表 `schema.sql`
- 修改数据库配置
- 跑起后端和前端

完成标志：

- 登录页可打开
- `admin/admin123` 可登录

### D2 代码生成与菜单准备

- 在若依代码生成器中导入业务表
- 生成菜品、分类、食材、库存、入库、订单基础 CRUD
- 合并代码到若依项目
- 配置业务菜单
- 配置三角色和测试账号

完成标志：

- 基础 CRUD 页面和接口可访问

### D3 菜品与分类

- 完成菜品分类 CRUD
- 完成菜品上下架
- 完成菜品页面筛选和弹窗联调

完成标志：

- 菜品管理模块可用

### D4 食材与配方

- 完成新增食材时初始化库存
- 完成配方查询和保存接口
- 完成食材页面
- 完成配方维护页面

完成标志：

- 菜品与食材建立可保存的配方关系

### D5 订单核心逻辑

- 完成模拟下单接口
- 完成完成订单接口
- 完成退单接口
- 验证库存不足时事务回滚

完成标志：

- 完成订单后库存按配方正确扣减

### D6 订单与库存前端

- 完成订单列表、详情、模拟下单弹窗
- 完成库存列表和入库弹窗
- 联调订单与库存变化

完成标志：

- 主业务闭环可前端演示

### D7 统计接口

- 完成今日营收统计
- 完成今日订单数统计
- 完成近 7 天营收趋势统计
- 完成销量 Top10 统计
- 完成库存预警列表接口

完成标志：

- 看板接口返回完整数据

### D8 看板前端

- 完成数字卡片
- 完成折线图
- 完成 Top10 柱状图
- 完成库存预警展示
- 调整系统标题为“文瀛餐厅管理系统”

完成标志：

- 核心页面全部可见且可演示

### D9 联调与演示数据

- 走完整业务链路
- 修复主流程问题
- 灌入近 7 天订单演示数据
- 检查角色权限隔离

完成标志：

- 系统稳定，演示数据完整

### D10 文档与答辩准备

- 完成部署说明
- 完成用户说明
- 完成演示脚本
- 准备答辩 PPT
- 至少彩排两次

完成标志：

- 软件、文档、演示材料齐全

## 3. 风险控制

- 如果配方页太复杂，简化为普通表单逐行录入
- 如果统计 SQL 时间过长，先保证 4 个核心指标
- 如果环境卡住，先优先解决 JDK、Maven、Node 版本
- 如果时间不足，优先保住“配方 -> 入库 -> 下单 -> 扣库存 -> 看板”链路

## 4. 最终验收清单

- 三角色登录正常
- 菜品增删改查正常
- 食材增删改查正常
- 配方维护正常
- 入库后库存增加
- 模拟下单成功
- 完成订单后库存扣减
- 库存不足时阻止完成订单
- 看板图表正常显示
- 安装说明可复现运行
