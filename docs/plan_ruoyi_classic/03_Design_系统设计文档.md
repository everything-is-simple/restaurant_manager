# 文瀛餐厅管理系统 — 系统设计文档

## 1. 整体架构

系统采用若依经典版前后端分离架构：

- 浏览器访问 Vue 2 后台前端
- 前端通过 `/dev-api` 代理访问 Spring Boot 后端
- 后端通过 JWT 认证与角色权限控制保护接口
- MySQL 存储业务数据
- Redis 提供登录态与缓存支持

## 2. 业务模块

| 模块 | 职责 |
|------|------|
| 菜品分类 | 管理分类字典 |
| 菜品管理 | 管理菜品基本信息与上下架状态 |
| 食材档案 | 管理食材与安全库存 |
| 配方管理 | 维护菜品与食材的用量关系 |
| 库存管理 | 查看库存、预警、执行入库 |
| 订单管理 | 模拟下单、查看订单、完成订单、退单 |
| 数据看板 | 展示统计数据与预警信息 |

## 3. 数据流

### 3.1 模拟下单

前端提交菜品和数量，后端校验菜品状态、计算金额、写入订单和订单明细。

### 3.2 完成订单扣库存

后端根据订单明细查找菜品配方，逐条计算食材扣减量，并在事务中更新库存。任何食材库存不足都应整体回滚。

### 3.3 入库

库管员录入入库记录后，系统新增一条 `t_stock_in` 记录，并同步累加 `t_inventory.stock`。

## 4. 数据库设计

业务表共 8 张：

- `t_category`
- `t_dish`
- `t_ingredient`
- `t_dish_ingredient`
- `t_inventory`
- `t_stock_in`
- `t_order`
- `t_order_item`

关系概览：

- 分类与菜品是一对多
- 菜品与食材通过配方表建立多对多
- 食材与库存是一对一
- 订单与订单明细是一对多

## 5. 工程落位

默认代码组织如下：

- 后端控制器：`ruoyi-admin/src/main/java/com/ruoyi/web/controller/restaurant`
- 后端实体、服务、Mapper：`ruoyi-system/src/main/java/com/ruoyi/system`
- Mapper XML：`ruoyi-system/src/main/resources/mapper/restaurant`
- 前端 API：`ruoyi-ui/src/api/restaurant`
- 前端页面：`ruoyi-ui/src/views/restaurant`

## 8. 核心实现点

### 8.1 扣库存逻辑

- 使用事务控制订单完成过程
- 根据订单明细循环查配方
- 每种食材扣减量 = 配方单份量 × 点单份数
- 更新库存时使用条件更新，避免扣成负数

### 8.2 统计接口

看板接口一次返回：

- 今日营收
- 今日订单数
- 近 7 天营收趋势
- 菜品销量 Top10
- 库存预警列表

### 8.3 配方维护页

该页面是唯一需要明显手写的前端页面，采用：

- 顶部选择菜品下拉框
- 下方表格动态编辑多行食材与用量
- 保存时覆盖写入配方

## 9. 异常处理

- 未登录返回 401
- 无权限返回 403
- 库存不足等业务错误抛 `ServiceException`
- 系统异常交由若依全局异常处理器统一响应

## 10. 演示数据

`schema.sql` 提供基础分类、菜品、食材、库存和样例配方。联调后再补充近 7 天订单数据，保证看板可视化效果稳定。
