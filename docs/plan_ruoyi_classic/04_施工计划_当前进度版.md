## 1. 项目定位

中北大学文瀛餐厅后台管理系统（课程/答辩项目），基于 **RuoYi-Vue 3.9.2 经典版**二次开发。

- 定位：高校食堂后台管理
- 不做：点餐小程序、在线支付、供应商管理
- 演示主闭环：`配方维护 → 食材入库 → 模拟下单 → 完成订单 → 按配方扣库存 → 看板统计`
- 答辩亮点：完成订单按菜品配方自动扣库存，使用 `UPDATE ... WHERE stock >= deduct` + 事务保证并发不超扣

## 4. 业务算法清单（按"扣库存闭环"主线排序）

| # | 算法 | 涉及表/模块 | 优先级 |
|---|------|------------|--------|
| 1 | 新增食材自动初始化库存（stock=0） | t_ingredient + t_inventory | P0 |
| 2 | 入库累加库存 | t_stock_in + t_inventory | P0 |
| 3 | 配方覆盖保存（按 dish_id 删除后批量插入） | t_dish_ingredient | P0 |
| 4 | 模拟下单（生成 order_no + 批量插入 order_item） | t_order + t_order_item | P0 |
| 5 | **完成订单按配方扣库存**（核心） | t_order + t_order_item + t_dish_ingredient + t_inventory | P0（答辩主菜）|
| 6 | 退单回滚库存 | t_order + t_order_item + t_dish_ingredient + t_inventory | P0 |
| 7 | 库存不足事务回滚保护 | t_inventory | P0 |
| 8 | 库存预警列表（stock < safety_stock） | t_inventory + t_ingredient | P1 |
| 9 | 看板统计（今日营收/订单数/7天趋势/Top10） | t_order + t_order_item + t_dish | P1 |

- 如果配方页太复杂，简化为普通表单逐行录入

- 库存扣减必须用 `UPDATE ... WHERE stock >= ?` 模式，禁止先 SELECT 再 UPDATE（避免并发超扣）
