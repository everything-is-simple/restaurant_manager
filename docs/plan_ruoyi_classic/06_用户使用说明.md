# 文瀛餐厅管理系统 — 用户使用说明

## 1. 登录入口

浏览器打开：`http://127.0.0.1`。

| 账号 | 密码 | 角色 | 主要职责 |
|------|------|------|----------|
| `admin` | `admin123` | 超级管理员 | 系统配置、角色菜单、餐厅全部模块 |
| `manager` | `admin123` | 食堂经理 | 菜品、配方、订单、库存查看、经营看板 |
| `stockkeeper` | `admin123` | 库管员 | 食材档案、入库登记、库存查看 |

## 2. 超级管理员

超级管理员用于系统初始化、菜单权限维护和答辩兜底检查。

常用操作：

- 在“系统管理”中维护用户、角色、菜单和字典。
- 在“餐厅管理”中查看全部业务菜单。
- 当角色菜单异常时，检查 MANAGER 和 STOCK_KEEPER 角色是否绑定对应菜单。

答辩中不建议长期使用 admin 演示业务链路，业务演示优先使用 `manager` 和 `stockkeeper`。

## 3. 食堂经理

食堂经理负责日常经营管理和答辩主链路演示。

可见菜单：

- 数据看板
- 菜品分类
- 菜品管理
- 库存管理
- 菜品配方
- 订单管理

主要操作：

1. 进入“菜品管理”，查看菜品分类、售价和上架状态。
2. 点击菜品行内“配方”，进入该菜品的配方维护页。
3. 在“菜品配方”中选择菜品，查看或维护食材用量。
4. 在“订单管理”中点击“模拟下单”，选择菜品和份数生成订单。
5. 对“已下单”订单点击“完成”，系统按菜品配方自动扣减库存。
6. 对需要回滚的订单点击“退单”，系统按原明细回滚库存。
7. 进入“数据看板”，查看今日营收、今日订单数、在售菜品、库存预警、近 7 天营收趋势和销量 Top10。

权限边界：食堂经理不直接维护食材档案和入库记录，避免经营角色越权修改库管基础数据。

## 4. 库管员

库管员负责食材基础资料、入库登记和库存状态检查。

可见菜单：

- 食材档案
- 库存管理
- 入库管理

主要操作：

1. 进入“食材档案”，新增或维护食材名称、单位和安全库存。
2. 新增食材后，系统会自动创建库存记录，初始库存为 0。
3. 进入“入库管理”，新增采购入库记录，系统自动增加库存。
4. 修改入库记录时，系统按新旧入库数量差额调整库存。
5. 删除入库记录时，系统回滚对应库存数量。
6. 进入“库存管理”，查看当前库存、安全库存和库存预警状态。

权限边界：库管员不参与订单完成、退单和经营看板查看。

## 5. 答辩主链路操作

推荐使用 `manager/admin123` 演示主链路：

1. 打开“菜品配方”，选择一道已上架菜品，展示该菜品需要哪些食材以及每份用量。
2. 打开“库存管理”，记住其中一种配方食材的当前库存。
3. 打开“订单管理”，点击“模拟下单”，选择刚才的菜品并填写份数。
4. 在订单列表中找到新订单，点击“完成”。
5. 回到“库存管理”，验证相关食材库存已按配方扣减。
6. 打开“数据看板”，验证今日营收和今日订单数已变化。
7. 如需展示回滚能力，对已完成订单点击“退单”，再回到库存页查看库存回滚。

## 6. 常见提示

- “库存不足”：说明某个配方食材当前库存小于订单所需扣减数量，订单不会完成，库存不会脏写。
- “无权限访问”：说明当前账号角色不具备该菜单或按钮权限，请切换到对应角色账号。
- 看板没有变化：看板按“已完成订单”的 `complete_time` 统计，只有完成订单后才进入经营数据。
- 配方为空：该菜品未维护配方，完成订单前应先补齐配方。
